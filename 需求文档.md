# K01 自动研判与数据分析需求文档（草案）

## 1. 背景与目标

基于输入文件 `/mnt/d/projects/k01/input/input.xlsx` 中的外联 IOC 数据，结合 xmon、文件情报、wfy、sc、wd 等接口结果及一系列判断规则，自动生成两个 Excel：

- `/mnt/d/projects/k01/output/result.xlsx`：K01 研判结果导入表，面向最终导入或交付。
- `/mnt/d/projects/k01/output/数据分析.xlsx`：覆盖度与可解决性分析表，面向规则核对、人工复盘和后续优化。

当前需求重点不是一次性把规则完全定死，而是先形成一版可执行、可核对、可逐步补充的需求基线。

## 3. 外部接口与查询数据

### 3.1 xmon IOC 查询

用途：

- 查询 IOC 在 xmon 中的主线索、子线索、禁用状态、存活状态、关联样本、关联报告等信息。

关键字段：

| 字段 | 用途 |
| --- | --- |
| `disable` / `Disabled` | 判断线索是否 disabled |
| `status` | 映射到 K01 存活状态 |
| `ref_sample` / `Raw.ref_sample` | 提取关联样本 hash |
| `report_links` / `Raw.report_links` | 提取关联报告链接 |
| src / source 相关字段 | 用于生产方归属判断 |

存活状态映射：

| xmon status | K01 存活状态 |
| --- | --- |
| `ACTIVE` | 存活 |
| `UNKNOWN` | 失活 |
| `SINKHOLE` | 被安全机构接管 |

### 3.2 文件情报接口

用途：

- 对 xmon 关联样本 hash 调用文件情报接口。
- 判断样本是否黑。
- 补充文件大小、文件类型、影响操作系统、创建时间、恶意家族等字段。

关键判断：

| 字段 | 规则 |
| --- | --- |
| `risk = high` 或其他黑判定值 | 认为关联样本可作为有效证据 |
| 无结果 / 白 / 未知 | 不命中黑样本规则 |

### 3.3 wfy reputation 接口

用途：

- 判断 IOC 是否被 wfy 识别为黑。
- 作为是否继续进入生产方归属判断的重要前置条件。

关键判断：

| wfy 结果 | 处理 |
| --- | --- |
| `judge = black` | 进入生产方归属与解决方案判断 |
| 无结果 / 未知 | 默认不继续走生产方归属深度判断，倾向标记为不可解决或无效，待确认 |

补充信息模板：

`wfy接口查询显示未知（超期或wfy不关心的情报），置无效，补info_add`

### 3.4 sc 接口

用途：

- 辅助生产方归属判断。
- 若 sc 查询结果为恶意，且未命中 atateam、siyubo、wd，则默认归属 netlab。

待确认：

- sc 接口入参、鉴权方式、返回字段、恶意判定字段。

### 3.5 wd / 内容快照智能体

用途：

- 判断 IOC 是否存在恶意快照。
- 对 `wd` 生产方归属提供证据。

当前已知规则：

- 归属为 `wd`，且 IOC 恶意，且存在快照，暂时标记为“能解决”。
- 话术模板：`内容类，存在恶意快照的ioc(xx主题内容)`

待确认：

- wd_v1 接口或脚本入口。
- 快照恶意判定字段。
- “主题内容”的提取来源。

### 4.4 补充信息生成规则

| 命中规则 | 补充信息模板 |
| --- | --- |

### 4.5 关联报告链接过滤规则

当前已知规则：

- 本地路径类报告不采信，例如：`/home/xuyang-pd/...`
- 带本地文件路径并追加时间戳的链接不采信，例如：`/home/xuyang-pd/qdev/maltrail/trails/static/malware/njrat.txt#2024-06-19_20:42:18`
- `raw.githubusercontent.com` 中满足可信路径的链接可考虑采信，例如包含 `/maltrail/trails/static/` 的链接，待确认。
- 普通外部报告链接是否采信需确认允许列表或域名白名单。

待确认：

- “允许的 list”具体清单。
- 是否只采信第一条合规报告。
- 报告链接格式化规则。

## 6. 生产方归属逻辑

### 6.1 总体流程

1. 对每条 IOC 调用 wfy 接口。
2. 如果 wfy 没有结果或结果不是黑，则不进入生产方归属深度判断，倾向归为 `unknown` 或不可解决。
3. 如果 wfy 判黑，则依次查询 xmon、sc、wd_v1 等来源。
4. 按固定优先级在 `atateam`、`siyubo`、`wd`、`netlab`、`unknown` 中确定一个最终归属。
5. 基于最终归属和证据命中情况，生成 `能否解决` 与 `相关解决方案`。

### 6.2 归属优先级

建议按以下顺序判定：

| 优先级 | 归属 | 判定规则 |
| --- | --- | --- |
| 1 | atateam | xmon 主线索、子线索中命中 atateam 相关关键词，例如 `apt.atateam` |
| 2 | siyubo | xmon 主线索、子线索中命中 siyubo 相关关键词，或命中 dga 关键词 |
| 3 | wd | wd_v1 显示 IOC 恶意 |
| 4 | netlab | sc 报恶意，且上面规则都未命中 |
| 5 | unknown | wfy 不报黑、无结果，或当前来源无法识别 |

### 6.3 xmon 线索有效性

当前图片中的判断口径：

- xmon 主线索恶意且非 disabled，认为可用。
- xmon 子线索恶意且非 disabled，认为可用。
- 如果点数数量在 `[0,1]`，可能是内部情报。
- 其他情况可能是开源情报。

待确认：

- “点数数量”的字段名。
- 主线索、子线索的接口字段名。
- “内部情报”和“开源情报”是否影响生产方归属，还是只影响可解决性。

## 7. 执行流程

建议实现流程如下：

1. 读取 `/mnt/d/projects/k01/input/input.xlsx`。
2. 标准化字段，生成统一 IOC。
3. 批量查询 xmon，提取 disabled、status、关联样本、关联报告、src 相关信息。
4. 对关联样本 hash 批量查询文件情报接口。
5. 执行 `result.xlsx` 研判规则，生成最终导入表。
6. 批量查询 wfy、sc、wd_v1 等接口。
7. 执行生产方归属逻辑。
8. 执行能否解决判断，生成 `数据分析.xlsx`。
9. 输出统计日志，包括总数、有效数、无效数、各规则命中数、各生产方归属数。

## 8. 当前已确认规则

- 厂商字段固定为 `360`。
- xmon `status` 映射为：`ACTIVE=存活`、`UNKNOWN=失活`、`SINKHOLE=被安全机构接管`。
- 关联黑样本可直接作为有效证据。
- 关联样本有多个时，当前暂取第一个。
- 关联报告有多个时，当前暂取第一个。
- 本地路径类报告暂不采信。
- 归属为 wd，且恶意且有快照，暂时标记为能解决。
- 数据分析表至少输出：`ioc`、`端口`、`厂商`、`生产方归属`、`能否解决`、`相关解决方案`。

## 10. 验收标准

- 能从 `/mnt/d/projects/k01/input/input.xlsx` 读取数据并稳定生成两个输出文件。
