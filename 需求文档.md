# K01 自动研判与数据分析需求文档

## 1. 当前目标

脚本基于 `input/input.xlsx` 和已生成的 `output/result.xlsx`，补齐两类输出：

- `output/result.xlsx`：研判结果表。当前字段基本已完成，本文档保留已确认规则，后续不再扩展无关逻辑。
- `output/数据分析.xlsx`：当前开发重点，用于分析每条 IOC 是否进入生产方归属判断、最终归属到哪个生产方、是否可解决，以及对应解决依据。

当前阶段不处理无关字段、不引入旧的误报/无效扩展逻辑。所有规则以本文档为准。

## 2. 输入与基础字段

### 2.1 输入文件

- 输入文件：`input/input.xlsx`
- 研判结果表：`output/result.xlsx`
- 数据分析表：`output/数据分析.xlsx`

### 2.2 IOC 生成规则

脚本内部用于接口查询的统一 `ioc` 按以下规则生成：

| 条件 | ioc 生成方式 |
| --- | --- |
| `目标类型 = IP` 且 `端口` 不为空 | `外联目标:端口` |
| `目标类型 = IP` 且 `端口` 为空 | `外联目标` |
| 非 IP 类型 | `外联目标` |

`result.xlsx` 和 `数据分析.xlsx` 中的 `ioc`、`端口`、`厂商` 保持一致。

## 3. result.xlsx 已确认规则

### 3.1 基础字段

| 字段 | 规则 |
| --- | --- |
| `IOC` / `ioc` | 与 result 表当前口径一致 |
| `端口` | 与输入或 result 表一致 |
| `厂商` | 固定为 `360` |
| `外联日期` | 来自输入表 |

### 3.2 存活状态

`存活状态` 根据 xmon 返回的 `status` 字段映射：

| xmon status | K01 存活状态 |
| --- | --- |
| `ACTIVE` | 存活 |
| `UNKNOWN` | 失活 |
| `SINKHOLE` | 被安全机构接管 |

如果 xmon 未返回 `status` 或返回值不在上述枚举内，第一阶段先置空，并在日志中记录待确认状态值。

### 3.3 补充信息与研判结果

`补充信息` 只填一个模板。先查 wfy，只有 wfy 返回 `judge = black` 时，才继续按 xmon 关联样本、xmon 关联报告、wd 恶意快照的顺序补证据。命中高优先级后，不再继续查低优先级证据。

| 优先级 | 查询来源 | 命中条件 | 补充信息模板 | 研判结果 |
| --- | --- | --- | --- | --- |
| 0 | wfy | wfy 查不到结果，或 `judge = white` | `wfy接口查询显示末知` | 空 |
| 1 | xmon `ref_sample` / `Raw.ref_sample` | wfy `judge = black`，且能提取到关联样本 hash | `{ioc}，依据ioc({ioc}),关联样本（{hash}）` | 有效 |
| 2 | xmon `report_links` / `Raw.report_links` | wfy `judge = black`，关联样本为空，且能提取到可用报告链接 | `{ioc}，依据ioc({ioc}),关联报告（{report_url}）` | 有效 |
| 3 | wd 接口 | wfy `judge = black`，关联样本和关联报告都为空，且存在恶意快照 | `内容类，存在恶意快照的ioc({主题内容})` | 有效 |
| 4 | 无 | wfy `judge = black`，但三类证据都未命中 | 空 | 空 |

示例：

- `buy1.mcloemb.ru，依据ioc(buy1.mcloemb.ru),关联样本（fa0018c8c9aaee50ec93e5d0bf7cac6f）`
- `buy1.mcloemb.ru，依据ioc(buy1.mcloemb.ru),关联报告（https://samples.vx-underground.org/APTs/2021/2021.10.20(1)/Paper/Bitdefender-DT-Whitepaper-Fivesys-creat5699-en-EN.pdf）`
- `内容类，存在恶意快照的ioc(xx主题内容)`
- `wfy接口查询显示末知`

关联报告过滤规则：

- `/home/xuyang-pd/...` 本地路径不采信。
- 本地路径加时间戳不采信。
- `raw.githubusercontent.com` 且包含 `/maltrail/trails/static/` 的链接可采信。
- 普通外部 URL 是否采信，后续按允许列表补充。

## 4. 数据分析.xlsx 字段规则

### 4.1 输出字段

当前数据分析表字段如下：

| 字段 | 规则 |
| --- | --- |
| `ioc` | 与 `result.xlsx` 的 IOC/ioc 口径一致 |
| `端口` | 与 `result.xlsx` 一致 |
| `厂商` | 与 `result.xlsx` 一致，固定为 `360` |
| `生产方归属` | 先查 wfy；wfy 查不到、unknown 或 `judge = white` 时填 `unknown`；wfy `judge = black` 时继续查 xmon、wd、netlab |
| `能否解决` | 根据 wfy 查询结果填写 |
| `相关解决方案` | 填入对应 `result.xlsx` 的 `补充信息` |

## 5. 数据分析处理流程

### 5.1 总体流程

对每条 result 数据执行：

1. 读取 `ioc`、`端口`、`厂商`，写入 `数据分析.xlsx`。
2. 读取同一条 result 的 `补充信息`，作为后续 `相关解决方案` 的候选值。
3. 调用 wfy 接口查询 IOC reputation。
4. 如果 wfy 查不到、返回 unknown 或 `judge = white`，`生产方归属` 填 `unknown`，不继续查询 xmon、wd、netlab。
5. 如果 wfy 返回 `judge = black`，继续查询 xmon、wd、netlab，汇总候选归属。
6. 根据 wfy 结果填写 `能否解决`。
7. 将 result 表对应行的 `补充信息` 写入 `相关解决方案`。

### 5.2 wfy 判断规则

wfy 用于判断 IOC 是否进入生产方归属判断，是数据分析表的前置条件。

接口示例：

```text
curl -X POST 'http://api.netlab.360.com/wfy/reputation/v1' \
  -H 'Content-Type: application/json' \
  -H 'X-AuthToken: 6189ff22-21c8-49ab-8519-7ef85f396954' \
  -d '["abvainvienvaiebai.net"]'

curl -X POST 'http://api.netlab.360.com/wfy/reputation/v1' \
  -H 'Content-Type: application/json' \
  -H 'X-AuthToken: 6189ff22-21c8-49ab-8519-7ef85f396954' \
  -d '["96.92.242.30:80"]'
```

| wfy 结果 | 是否继续生产方归属判断 | 生产方归属 | 能否解决 |
| --- | --- | --- | --- |
| `judge = black` | 是 | 继续按 xmon、wd、netlab 判断 | 是 |
| `judge = white` | 否 | `unknown` | 否 |
| 查不到 | 否 | `unknown` | 否 |
| 未知 / unknown | 否 | `unknown` | 否 |

说明：

- wfy 返回字段 `judge = black` 表示该 IOC 是黑样品，继续查 xmon、wd、netlab，汇总候选归属后再按优先级选择最终 `生产方归属`。
- wfy 返回 `judge = white`、查不到结果或返回 unknown 时，`生产方归属` 直接填 `unknown`，不再查 xmon、wd、netlab。
- wfy 返回 `judge = white` 或查不到结果时，`result.xlsx` 的 `研判结果` 置空，`补充信息` 填 `wfy接口查询显示末知`。

## 6. 生产方归属规则

### 6.1 数据来源

生产方归属由多个来源共同判断：

| 来源 | 用途 |
| --- | --- |
| wfy | 前置判断；`judge = white`、查不到或 unknown 时直接归属 `unknown` |
| xmon 主线索 / 子线索 | 只用于判断 `atateam`、`siyubo` |
| wd 接口 | 有数据时追加候选归属 `wd` |
| netlab 库接口 | 有数据时追加候选归属 `netlab` |

最终把所有候选归属放入一个 list，再按 `atateam > siyubo > wd > netlab > unknown` 的优先级选择一个填入 `生产方归属`。

### 6.2 xmon 线索处理

接口配置来自 `k01_run_process.py`：

```text
GET http://xmon.netlab.qihoo.net/api/iocmon-search/ioc/{ioc_part}/?ui_simple=true&pretick=false&model=false&trace=true&keep_no=false&inspect=true&other_source=false
```

其中：

- `{ioc_part}` 为批量 IOC 查询字符串，多个 IOC 用英文逗号拼接。
- IOC URL 编码时保留 `.:_-` 字符。
- 当前脚本每批最多查询 10 条 IOC。
- 请求超时时间：60 秒。
- xmon 查询失败时记录错误，不中断整批任务。

请求头：

```text
Accept: */*
Accept-Language: zh-CN,zh;q=0.9
Cache-Control: no-cache
Connection: keep-alive
X-AuthToken: 6189ff22-21c8-49ab-8519-7ef85f396954
Pragma: no-cache
Referer: http://xmon.netlab.qihoo.net/ui/iocmon/
User-Agent: Mozilla/5.0
X-Requested-With: XMLHttpRequest
```

脚本当前从 xmon 返回中识别 IOC 的字段优先级：

```text
ioc_search -> ioc -> IOC -> uid
```

需要关注字段：

| 字段 | 说明 |
| --- | --- |
| 主线索 | xmon 返回的主线索对象 |
| 子线索 | xmon 返回的子线索 list |
| `disable` | 判断该线索是否需要忽略 |
| `src` | 用于判断是否命中 `atateam`、`siyubo` 的字符串 |

对 xmon 返回的主线索和子线索逐条处理，沿用之前逻辑：

1. 如果该线索的 `disable` 字段有任何值，则忽略这一条线索，包括 `False`、`0`、`"false"` 等值。
2. 如果该线索的 `disable` 字段为空，则读取该线索的 `src` 字段。
3. 将所有有效线索的 `src` 汇总成一个 list。
4. `src` 当前接口返回均为小写；实现时仍统一转小写后再做包含判断，只匹配 `atateam`、`siyubo`。
5. 如果命中 `atateam` 或 `siyubo`，将命中的归属追加到候选归属 list。

### 6.3 wd 判断

在 wfy 返回 `judge = black` 后，继续查询 wd 接口：

- 如果 wd 接口有数据，则将 `wd` 追加到候选归属 list。
- 如果 wd 接口无数据，则不追加 `wd`。

### 6.4 netlab 判断

在 wfy 返回 `judge = black` 后，继续通过接口查询 netlab 库：

- 如果 netlab 库有数据，则将 `netlab` 追加到候选归属 list。
- 如果 netlab 库无数据，则不追加 `netlab`。

### 6.5 归属关键词与优先级

候选归属来源：

| 归属 | 来源 / 匹配方式 |
| --- | --- |
| `atateam` | xmon 有效线索的 `src` 字符串包含 `atateam` |
| `siyubo` | xmon 有效线索的 `src` 字符串包含 `siyubo` |
| `wd` | wd 接口有数据 |
| `netlab` | netlab 库接口有数据 |
| `unknown` | wfy `judge = white`、查不到 / unknown，或所有来源都未命中 |

如果候选 list 中有多个归属，只填一个最终 `生产方归属`，按以下优先级选择：

```text
atateam > siyubo > wd > netlab > unknown
```

示例：

| 候选归属 list | 最终生产方归属 |
| --- | --- |
| `["netlab", "wd"]` | `wd` |
| `["siyubo", "netlab"]` | `siyubo` |
| `["atateam", "wd", "netlab"]` | `atateam` |
| `[]` 或无关键词命中 | `unknown` |

## 7. 数据分析字段填充细则

### 7.1 ioc、端口、厂商

| 字段 | 填充方式 |
| --- | --- |
| `ioc` | 与 `result.xlsx` 对应行一致 |
| `端口` | 与 `result.xlsx` 对应行一致 |
| `厂商` | 与 `result.xlsx` 对应行一致，固定为 `360` |

### 7.2 生产方归属

| 条件 | 生产方归属 |
| --- | --- |
| wfy `judge = white` / 查不到 / unknown | `unknown` |
| wfy `judge = black`，且候选归属 list 非空 | 按 `atateam > siyubo > wd > netlab > unknown` 优先级填一个 |
| wfy `judge = black`，但 xmon、wd、netlab 都未命中 | `unknown` |

### 7.3 能否解决

| 条件 | 能否解决 |
| --- | --- |
| wfy `judge = black` | 是 |
| wfy `judge = white` / 查不到 / unknown | 否 |

### 7.4 相关解决方案

`相关解决方案` 直接填入对应 `result.xlsx` 行的 `补充信息`。

如果 result 表对应行 `补充信息` 为空，则 `相关解决方案` 也置空。

## 9. 验收标准

- 能读取 `output/result.xlsx` 并生成 `output/数据分析.xlsx`。
- `数据分析.xlsx` 的 `ioc`、`端口`、`厂商` 与 result 对应行一致。
- 每条数据都先查 wfy。
- wfy `judge = white`、查不到或 unknown 的数据直接归属 `unknown`，不再查询 xmon、wd、netlab。
- wfy `judge = black` 的数据才继续进入 xmon、wd、netlab 生产方归属判断。
- xmon 主线索和子线索中 `disable` 不为空的线索必须被忽略。
- `生产方归属` 只填一个值，优先级为 `atateam > siyubo > wd > netlab > unknown`。
- `能否解决` 根据 wfy 查询结果填写。
- `相关解决方案` 来自 result 表对应行的 `补充信息`。
