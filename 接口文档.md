下面是更改后的完整版本：**`error` 放到 `data[]` 每个 IOC 结果对象的同级字段里**，不再放到 `project_k01_table_daily` 里面。这样批量场景下，调用方可以直接通过 `data[i].error` 判断当前 IOC 是否处理异常。

# K01 情报研判接口需求文档

## 1. 接口说明

该接口用于对 IOC 进行情报研判。

接口只提供一个能力：**批量 IOC 研判**。

本接口不负责：

* Excel 导入导出
* 日志查询
* hourly 表
* 情报反馈需求接收
* 异步任务
* 任务状态查询
* 结果分页查询

接口返回每个 IOC 对应的研判数据，主要包括：

* `ioc`：当前结果对应的原始 IOC
* `error`：当前 IOC 的处理错误信息，正常为空字符串
* `sc`：SC 情报查询结果
* `agent`：智能体研判结果
* `project_k01_table_daily`：K01 日表最终输出结果

# 3. 输入参数

## 3.1 输入说明

接口输入参数包含 `ioc`，以及可选的 `project`：

```json
{
  "project": "project_k01_table_daily",
  "ioc": []
}
```

其中 `ioc` 是已经拼接好的 IOC 列表。
`project` 是可选字段。只有当 `project` 等于 `project_k01_table_daily` 时，响应才返回 `project_k01_table_daily`；不传或传其他值时，响应只返回 `sc` 和 `agent`。

也就是说，调用方在传入前需要自己处理好 IOC 格式。

接口内部不再接收以下字段：

```text
port
ioc_type
vendor
out_date
```

接口支持批量传入。即使只查询一个 IOC，也需要用数组传入。

---

| `project` | String | 否 | `project_k01_table_daily` | 响应字段开关。等于该值时返回 `project_k01_table_daily`，不填或其他值时不返回 |
| `ioc` | Array[String] | 是  | `["maimai666.com", "96.92.242.30:80", "http://example.com/a/b"]` | 待研判 IOC 列表，支持批量。IOC 必须是已经拼接好的完整值 |

# 4. 校验规则

## 4.1 校验原则

由于该接口是批量接口，所以校验分为两类：

```text
1. 请求级校验：请求整体格式错误，整个接口失败，data 返回空数组
2. 单条 IOC 校验：某一条 IOC 错误，只影响当前 IOC，其他 IOC 正常处理并返回
```

---

## 4.2 请求级校验

请求级校验失败时，接口整体失败，返回 `data: []`。

| 校验项         | 规则                | 失败影响范围 | 失败返回         |
| ----------- | ----------------- | ------ | ------------ |
| 请求体不能为空     | 请求体必须是合法 JSON     | 整体失败   | `请求体不能为空`    |
| `ioc` 必须存在  | 请求体中必须包含 `ioc` 字段 | 整体失败   | `缺少必填字段：ioc` |
| `ioc` 必须是数组 | 不允许传字符串、对象、数字     | 整体失败   | `ioc 必须是数组`  |
| 服务端异常       | 服务端运行异常           | 整体失败   | `服务端异常`      |

---

## 4.3 单条 IOC 校验

单条 IOC 校验失败时，不影响其他 IOC。

| 校验项          | 规则                             | 失败影响范围   | 返回说明                                                |
| ------------ | ------------------------------ | -------- | --------------------------------------------------- |
| `ioc` 数组为空   | 如果 `ioc` 数组为空，服务端直接返回空 `data`  | 整体成功，空结果 | `code=0`，`data=[]`                                  |
| 单条 IOC 为空    | 当前 IOC 返回错误，不影响其他 IOC          | 单条失败     | `data[].error = "IOC 不能为空"`                         |
| 单条 IOC 类型不支持 | 必须是 `domain`、`ip:port` 或 `url` | 单条失败     | `data[].error = "不支持的ioc类型（请输入domain，ip:port，url）"` |
| IOC 前后空格     | 服务端自动 `trim`                   | 不失败      | 继续处理                                                |

---

## 4.4 SC 情报校验

SC 情报校验属于单条 IOC 处理流程，不影响其他 IOC。

| SC 查询结果   | 处理规则       | 返回说明                                                                                |
| --------- | ---------- | ----------------------------------------------------------------------------------- |
| SC 接口报恶意  | 继续走智能体研判流程 | 在 `data` 中返回 `sc` 查询结果、`agent` 查询结果和 `project_k01_table_daily` 结果，`data[].error` 为空 |
| SC 未查询到   | 不继续走智能体研判  | `data[].error` 和 `supplement_info` 返回 `非360情报（ti平台查询：https://ti.360.cn/）`           |
| SC 查询结果为白 | 不继续走智能体研判  | `data[].error` 和 `supplement_info` 返回 `无对应智能体研判结果，请重试`                              |

---

## 4.5 数量统计规则

| 字段              | 统计规则                                |
| --------------- | ----------------------------------- |
| `total`         | 请求中 IOC 的总数量                        |
| `success_count` | `data[].error` 为空，且完整完成研判流程的 IOC 数量 |
| `failed_count`  | `data[].error` 非空的 IOC 数量           |

示例：

```json
{
  "ioc": [
    "maimai666.com",
    "96.92.242.30",
    "96.92.242.30:80"
  ]
}
```

如果：

```text
maimai666.com          正常
96.92.242.30           不支持，因为单独 IP 不允许
96.92.242.30:80        正常
```

则返回统计为：

```json
{
  "total": 3,
  "success_count": 2,
  "failed_count": 1
}
```

---

## 4.6 code 和 message 规则

| 场景                  |    code | message           | data                                 |
| ------------------- | ------: | ----------------- | ------------------------------------ |
| 全部 IOC 正常完成         |     `0` | `success`         | 返回全部结果                               |
| 部分 IOC 正常，部分 IOC 失败 |     `0` | `partial success` | 返回所有 IOC 的结果，失败项在 `data[].error` 中说明 |
| 请求体为空               | `40001` | `请求体不能为空`         | `[]`                                 |
| 缺少 `ioc` 字段         | `40002` | `缺少必填字段：ioc`      | `[]`                                 |
| `ioc` 不是数组          | `40003` | `ioc 必须是数组`       | `[]`                                 |
| 服务端异常               | `50000` | `服务端异常`           | `[]`                                 |

注意：
单条 IOC 类型不支持，不返回整体 `40004`，而是放入当前 IOC 对应的 `data[].error` 中，其他 IOC 继续正常处理。

## 6.10 other_evidence 字段说明

`other_evidence` 表示除样本行为、流量片段、钓鱼信息、参考链接以外的其他证据。

| 字段名                   | 类型            | 是否必返 | 填写规范与示例                                           | 业务含义                         |
| --------------------- | ------------- | ---- | ------------------------------------------------- | ---------------------------- |
| `parent_intelligence` | Array[String] | 是    | `["evil.com", "1.2.3.4"]`                         | 该 IOC 的父情报、上级情报或关联情报         |
| `parent_evidence`     | String        | 是    | `父情报中提及该 IOC 与恶意样本通信，关联 hash_md5、hash_sha256 等字段` | 父情报中的关联依据或上下文说明              |
| `pivoting_feature`    | String        | 是    | `同域名注册邮箱 admin@evil.com，关联发现该邮箱注册的其他恶意域名`         | 扩线特征，例如关联域名、关联样本、关联 IP、注册邮箱等 |

## 6.11 project_k01_table_daily 字段说明

| `port`                      | String | 是    | `80`；如果 IOC 已经是 `ip:port`，该字段可为空                   | 端口                       |
| `vendor`                    | String | 是    | `360`                                              | 厂商                       |
| `outbound_date`             | String | 是    | `2026-06-30`；没有则为空字符串                              | 外联日期                     |
| `judgement_result`          | String | 是    | `恶意`、`可疑`、`误报`；无结果时为空字符串                           | 最终研判结果                   |
| `alive_status`              | String | 是    | `存活`、`失活`、空字符串                                     | IOC 存活状态                 |
| `file_hash`                 | String | 是    | `e99a18c428cb38d5f260853678922e03`                 | 文件特征值，可填 MD5、SHA1、SHA256 |
| `file_size`                 | String | 是    | `48.57 KB (49735 bytes)` 或 `49735`                 | 文件大小                     |
| `file_type`                 | String | 是    | `PE32 executable (GUI) Intel 80386 for MS Windows` | 文件类型                     |
| `affected_operating_system` | String | 是    | `windows`、`linux`、`android`                        | 影响操作系统                   |
| `create_time`               | String | 是    | `2026-06-30 10:00:00`                              | 文件创建时间、首次发现时间或情报时间       |
| `related_process`           | String | 是    | `2396 - C:\\Windows\\System32\\conhost.exe`        | 相关进程 ID 及文件名称            |
| `icp_connection_record`     | String | 是    | `192.168.1.100:4444`                               | ICP/TCP 连接记录             |

# 10. 业务状态码

| code    | message           | 说明                       |
| ------- | ----------------- | ------------------------ |
| `0`     | `success`         | 请求成功，所有 IOC 均正常完成        |

# 11. 最终说明

本接口当前只做一件事：

```text
批量接收已经拼接好的 IOC，并返回每个 IOC 的 SC 查询结果、智能体证据链结果和 K01 日表最终研判结果。
```

批量接口的核心原则是：

```text
请求结构错误：整体失败，data=[]
单条 IOC 错误：只失败当前 IOC，data[] 中保留该 IOC 的错误结果
其他正常 IOC：继续正常研判并返回结果
```

单条 IOC 是否处理成功，统一看：

```json
{
  "error": ""
}
```

判断规则：

| `data[].error` | 含义                                         |
| -------------- | ------------------------------------------ |
| 空字符串           | 当前 IOC 无处理错误，正常完成研判流程                      |
| 非空字符串          | 当前 IOC 处理失败、校验失败或无法继续研判，错误原因就是 `error` 的内容 |

本期不处理：

```text
Excel 导入导出
project_k01_api_hourly
log 查询
情报反馈需求接收
异步任务
任务状态查询
结果分页查询
```
